\subsection{Detailed Description}
The String \doxyref{construct}{p.}{classcubescript_1_1construct} is a character sequence enclosed in double quotes, and is similar to the \doxyref{word}{p.}{classcubescript_1_1cons_1_1word} \doxyref{construct}{p.}{classcubescript_1_1construct} in that its evaluation returns the parse input, excluding the double quotes. The \doxyref{string}{p.}{classcubescript_1_1cons_1_1string} \doxyref{construct}{p.}{classcubescript_1_1construct} is useful for supporting \doxyref{string}{p.}{classcubescript_1_1cons_1_1string} literals, which may contain characters that would cause syntax interpretation. Parsing is terminated by a double-quote or newline character.

An extension to the CubeScript language, not found in Sauerbraten's implementation of CubeScript, is support for the escape character. The escape character (\char`\"{}$\backslash$\char`\"{}) allows a double quote character to be expressed literally, any character proceeding a blackslash is treated literally.

\begin{Desc}
\item[Note:]Escape sequences are not supported, this is using metacharacters to generate characters (e.g. \char`\"{}$\backslash$n\char`\"{}). \end{Desc}
