\subsection{Detailed Description}
The \doxyref{expression}{p.}{classcubescript_1_1cons_1_1expression} \doxyref{construct}{p.}{classcubescript_1_1construct} is the core execution \doxyref{construct}{p.}{classcubescript_1_1construct} of the scripting language, an \doxyref{expression}{p.}{classcubescript_1_1cons_1_1expression} represents an apply operation to be performed on a \doxyref{symbol}{p.}{classcubescript_1_1symbol} object, the meaning of apply is dependant on the \doxyref{symbol}{p.}{classcubescript_1_1symbol} object's type. A CubeScript programmer writes a script from a sequence of expressions; no distinction is made between statements and expressions, the engine treats all expressions as one type of thing: an apply operation. Apply operations are abstract to the \doxyref{expression}{p.}{classcubescript_1_1cons_1_1expression} evaluator, only the script writer knows what apply operation is performed, by the \doxyref{symbol}{p.}{classcubescript_1_1symbol} name given.

An \doxyref{expression}{p.}{classcubescript_1_1cons_1_1expression} consists of elements separated by whitespace characters, with the first element being the \doxyref{symbol}{p.}{classcubescript_1_1symbol} name and the remaining elements being the arguments. An \doxyref{expression}{p.}{classcubescript_1_1cons_1_1expression} is terminated by a newline or semicolon character. An element can be a \doxyref{word}{p.}{classcubescript_1_1cons_1_1word}, \doxyref{string}{p.}{classcubescript_1_1cons_1_1string}, \doxyref{block}{p.}{classcubescript_1_1cons_1_1block} or \doxyref{expression}{p.}{classcubescript_1_1cons_1_1expression}. When the element constructs have been evaluated, from left to right, the apply method will be called on the \doxyref{symbol}{p.}{classcubescript_1_1symbol} object. The result value of the apply operation is the result of the \doxyref{expression}{p.}{classcubescript_1_1cons_1_1expression}.

There is an \doxyref{exception}{p.}{classcubescript_1_1exception} to the rule of expected syntax: the \doxyref{expression}{p.}{classcubescript_1_1cons_1_1expression} \doxyref{construct}{p.}{classcubescript_1_1construct} can also detect and accept an \doxyref{alias}{p.}{classcubescript_1_1alias} assignment using infix notation (i.e. \char`\"{}name = [value]\char`\"{}), for convenient use.

Parse CubeScript code. 

\begin{Desc}
\item[Returns:]The return value is True when parsing is complete, otherwise False is the return value.\end{Desc}
The parse method is stream-oriented; when the current function call cannot complete the parsing, because not enough input has been given yet, the function will return to the caller, who will be responsible to re-call the function when more input data is received - the parsing state is preserved among multiple function calls until parsing is complete.

When a parsing error is detected an \doxyref{exception}{p.}{classcubescript_1_1exception} of parse\_\-error type is thrown.
